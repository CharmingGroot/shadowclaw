# ShadowClaw PRD (Product Requirements Document)

OpenClaw 설계를 참고하여, **채팅 중심**의 개인 AI 어시스턴트 제품 요구사항 정리.

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| **제품명** | ShadowClaw |
| **정의** | 사용자 명령을 OpenClaw와 동일한 일처리 방식(ReAct·도구 호출·세션)으로 수행하는, 채팅이 메인인 개인 AI 어시스턴트 |
| **핵심 원칙** | 메인 화면 = 채팅, **소스 연결**(문서·URL·텍스트)로 답변 품질 향상, MCP·Skill 자유 연동·관리, API Key 안전 관리 |

---

## 2. 기술 스택

| 항목 | 내용 |
|------|------|
| **언어** | **TypeScript** (정적 타입, 유지보수성). |
| **런타임·서버** | **Node.js** (에이전트 API, ReAct, MCP 클라이언트, 세션·도구 관리). |
| **외부 MCP** | **외부 MCP 서버 연결** 지원. 사용자가 도구 화면에서 MCP 서버(URL 또는 stdio)를 등록·연결·해제하고, 등록된 서버의 도구를 채팅 시 에이전트가 호출할 수 있도록 기획. → [menu/tools/02-mcp.md](../menu/tools/02-mcp.md), [menu/tools/04-api.md](../menu/tools/04-api.md) |

## 3. OpenClaw 설계 분석 요약

ShadowClaw가 따를 OpenClaw의 핵심 설계는 다음과 같다.

| 영역 | OpenClaw 설계 | ShadowClaw 반영 |
|------|----------------|-----------------|
| **제어 구조** | Gateway(제어면) + Pi agent(RPC) + 채널·클라이언트 | 단일 서버(또는 Gateway 유사) + 에이전트 런타임, 채팅 클라이언트가 메인 |
| **일처리** | 사용자 메시지 → 에이전트가 도구/스킬 호출 → 스트리밍 응답, HITL·재시도 정책 | **동일**: 사용자 명령 → ReAct 루프(사고·도구 선택·실행·반복) → 응답 |
| **도구·스킬** | Skills 플랫폼(bundled/managed/workspace), ClawHub 레지스트리, MCP는 mcporter 브릿지 | **MCP와 Skill을 자유롭게 연동·조작·관리**: 등록/해제/설정 UI, MCP 서버 연결, 스킬 목록·상세·강제 호출 |
| **세션** | Session 모델(main, 그룹 격리, 활성화/큐 모드) | 세션별 대화 이력, 세션 전환·생성·삭제·제목 수정 |
| **보안·인증** | 페어링, allowlist, 모델/채널별 인증 | **API Key를 개인설정 메뉴에서 입력·저장**, 안전한 관리(암호화·미노출 표시) |
| **채널** | WhatsApp, Telegram, WebChat 등 다채널 | 1차: **WebChat(메인 화면 = 채팅)**. 확장 시 채널 추가 |

## 4. 핵심 요구사항 (Must Have)

### 4.1 메인 화면 = 채팅

- 앱 진입 시 **기본 라우트는 채팅 화면**.
- 채팅 화면 구성: 세션 목록(좌측 또는 접이식), 대화 영역(중앙), 필요 시 우측 패널(스킬/도구 목록·선택). **소스(Sources)** 패널 또는 탭으로 연결된 문서·URL·텍스트를 관리(노트북LM 스타일 참고).
- 사용자가 메시지를 입력·전송하면 **OpenClaw와 동일한 일처리 흐름**으로 에이전트가 응답(ReAct, 도구 호출, 스트리밍 가능). 연결된 소스가 있으면 해당 콘텐츠를 컨텍스트로 활용해 답변.

### 4.1.1 소스 연결 (Source Connection, 노트북LM 참고)

- **역할**: 사용자가 참조할 문서·웹·텍스트를 "소스"로 등록하고, 채팅 시 에이전트가 해당 소스를 활용해 답변하도록 함.
- **소스 유형(목표)**:
  - **파일 업로드**: PDF, 텍스트, 이미지, 오디오 등. 업로드 시 텍스트 추출 또는 메타만 저장 후 검색/참조.
  - **웹 URL**: 웹사이트 URL 입력 시 스크래핑 또는 요약을 소스로 저장.
  - **텍스트 붙여넣기**: 사용자가 직접 붙여넣은 텍스트를 소스로 등록.
  - (확장) 웹 검색 결과, 외부 스토리지(Drive 등) 연동은 별도 기획.
- **UI 요건**: 채팅 좌측 또는 별도 "소스" 영역에서 "소스 추가" 버튼, 소스 목록(이름·유형·삭제), 선택된 소스를 현재 세션/채팅에 연결.
- **연동**: 세션 또는 글로벌 단위로 "이 대화에 사용할 소스" 지정. ReAct/LLM 호출 시 지정된 소스 내용을 컨텍스트로 포함.
- 상세: [menu/sources/01-overview.md](../menu/sources/01-overview.md), [menu/sources/02-api.md](../menu/sources/02-api.md).

### 4.2 MCP와 Skill 연동·조작·관리

- **MCP**
  - MCP 서버를 등록·연결·해제할 수 있어야 함.
  - 등록된 MCP 서버의 **도구(Tools)** 목록을 조회·노출.
  - 채팅 시 에이전트가 MCP 도구를 호출할 수 있어야 함(서버 설정 시).
- **Skill**
  - 내장/등록 스킬 목록 조회, 스킬별 설명·파라미터 스키마 노출.
  - 스킬 활성/비활성, 필요 시 "이번 질의에 사용할 스킬" 강제 지정(채팅 UI).
- **연동**
  - MCP 도구와 Skill을 동일한 "도구" 풀에서 관리·노출할 수 있도록 설계(통합 목록 또는 구분 표시).

### 4.3 OpenClaw와 동일한 일처리 방식

- 사용자 명령 수신 → **ReAct 스타일 루프**: 생각(Thought) → 도구 선택(Action) → 도구 실행 → 관찰(Observation) → 반복 또는 최종 응답.
- 도구 실행 시: 내부 Skill 호출 또는 **MCP 클라이언트를 통한 MCP 도구 호출**.
- HITL(Human-in-the-Loop): 위험/중요 액션은 사용자 승인 후 실행(선택 사양).
- 스트리밍: 토큰/블록 스트리밍 지원 권장.

### 4.4 API Key — 환경변수만 사용

- **API Key는 UI에서 입력·관리하지 않음.** (클라이언트 전달·localStorage 저장 시 노출 위험.)
- 서버 실행 시 **환경변수**로만 설정: `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`.
- 설정 메뉴에서는 **모델(프로바이더) 선택**만 가능. API Key 미설정 시 스텁 응답 및 안내 메시지.

## 5. 제안 메뉴 구조

| 메뉴 | 설명 | 비고 |
|------|------|------|
| **채팅 (Chat)** | 메인 화면. 세션 목록 + 대화 + 스킬/도구 패널. 소스 연결 시 해당 소스 기반 답변. 사용자 명령 → OpenClaw 방식 일처리. | 기본 라우트 |
| **소스 (Sources)** | 노트북LM 스타일. 문서·URL·텍스트 소스 추가/삭제/목록, 세션별 또는 글로벌 "사용할 소스" 지정. | 답변 품질·참조용 |
| **도구 (Tools)** | MCP·Skill 연동·조작·관리. MCP 서버 등록/해제, 도구 목록, 스킬 목록·설정·활성화. | MCP·Skill 자유 연동 요건 |
| **설정 (Settings)** | API Key 입력·저장·안전 관리, 모델 선택, 기타 앱 설정. | 필수 |
| **세션 (Sessions)** | (선택) 세션 전용 목록·검색·삭제·제목 수정. 채팅 좌측에 있으면 생략 가능. | 채팅에 통합 가능 |
| **공통 (Common)** | 네비게이션, 반응형, 빈/에러 상태, 디자인 톤. | 모든 화면 공통 |

상세 화면별·기능별 명세는 **docs/menu/** 하위에 화면별 디렉토리와 기능별 명세 파일로 작성한다.

## 6. 비기능 요구사항

- **보안**: API Key·토큰은 암호화 저장, 전송 시 HTTPS.
- **프라이버시**: 대화·세션 데이터는 사용자 환경 내 저장(클라우드 동기화는 별도 요구 시).
- **호환성**: OpenClaw의 "Pi agent + 도구 + 세션" 개념과 정신적으로 호환되는 일처리 모델 유지.

## 8. 정리

- **메인 화면**: 채팅.
- **MCP·Skill**: 자유롭게 연동·조작·관리(도구 메뉴 + 채팅 우측 패널).
- **일처리**: 사용자 명령 시 OpenClaw와 동일한 ReAct·도구 호출 방식.
- **API Key**: 개인설정 메뉴에서 입력·안전 관리.
- **기획 메뉴**: 채팅, **소스**, 도구, 설정, (세션), 공통 — 화면별 docs/menu 하위에 기능별 명세로 정리.
