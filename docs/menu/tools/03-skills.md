# 도구 — 스킬

## 기능

- **사용자 관리 스킬** 목록 조회·설정. **내장 Skill은 UI에 보여주지 않음** (에이전트가 내부적으로만 사용).
- 스킬별 설명·파라미터 스키마 노출(목록에 노출된 것만).
- 스킬 활성/비활성 설정.
- **스킬 편집**: 스킬을 마크다운(md) 형태로 관리. 사용자가 md 본문을 직접 수정해 스킬 설명·내용을 관리.
- 채팅 우측 패널·강제 지정과 연동.

## 스킬 등록·편집·삭제 (이중 경로)

- **사용자 경로**: 도구 메뉴에서 **스킬 등록** 버튼으로 새 스킬을 직접 등록. **이름 + 마크다운 본문(.md)** 만 입력. 설명·params_schema는 본문에서 자동 파싱(첫 문단 = 설명, `## params_schema` + JSON 블록 = 파라미터). 등록된 스킬은 목록에서 **편집**(md)·**삭제** 가능.
- **에이전트 경로**: ReAct 에이전트가 **내장 도구**로 스킬을 등록·편집·삭제할 수 있어야 함. 따라서 다음 내장 스킬이 필수이다.
  - **등록**: `create_custom_skill` — **name + content(마크다운)** 권장. content 없으면 description·params_schema 인자로만 등록 가능(하위 호환).
  - **편집**: `update_skill_meta` — 설명, require_hitl, enabled, **content**(md 본문) 수정.
  - **삭제**: `delete_custom_skill` — 커스텀 스킬만 삭제 가능(내장 스킬은 삭제 불가).
- 두 경로는 동일한 저장소(레지스트리·오버라이드)를 사용하며, UI 목록과 에이전트 호출이 일치한다.

## 스킬 = 마크다운 관리

- 스킬은 결국 **마크다운 문서**로 표현 가능(이름, 설명, 파라미터, 본문).
- 사용자가 해당 md 내용을 직접 편집하여 스킬 메타·설명을 관리할 수 있어야 함.
- 저장 시 서버에 반영(오버라이드 또는 파일 저장은 구현 정책에 따름).

## UI 요건

- **내장 스킬 비노출**: 도구 화면·채팅 우측 스킬 패널에는 **사용자 정의(커스텀) 스킬만** 표시. 내장(read_file, write_file, list_dir 등)은 목록에서 제외.
- **스킬 등록**: 스킬 섹션 상단에 **「스킬 등록」** 버튼. 클릭 시 **모달**에서 **이름(필수)** + **마크다운 본문(.md)** 입력 후 등록. 스킬은 md 단일 소스로만 관리되며, 설명·params_schema는 본문에서 파싱(첫 문단, `## params_schema` JSON 블록). 등록 후 목록 새로고침.
- 목록, 상세, 토글(활성/비활성) — 위 목록에 한함.
- **스킬 편집**: 목록 행마다 **「편집」** 버튼. 클릭 시 **모달(팝업)** 을 띄우고, 모달 안에 해당 스킬의 **마크다운 본문**을 텍스트 영역으로 표시·편집. [취소] / [저장]으로 닫기 또는 저장 후 닫기. 인라인 입력 없음.
- **스킬 삭제**: 목록 행마다 **「삭제」** 버튼. 커스텀 스킬만 삭제 가능(확인 후 삭제).
